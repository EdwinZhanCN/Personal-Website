Fake Class

Initially, I do not have any concept of class. These dictionary named by card1, card2, card3... just behaves like a real card, because it has some properties like "isMatched", "isFaceUp"...These properties compose of each card, so that the game can be continued. At that time, It is the best way for me to deal with the matching and changing state of each card.

After I learned java, the real Class showed me that what I did is stupid... hhhh

Game Logic

With this "Fake Class", the logic of the game is very simple. I prepared a list to store the cards clicked by the user, and then compared the "CardContent" of the two cards. Finally, I use the output to determine whether to flip (change the color) or not.
